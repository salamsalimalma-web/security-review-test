OAuth2 Token Refresh Race Conditions in Multi-Tab Browser Sessions

Dr. Sarah Chen, Prof. Michael Torres, Dr. Anil Gupta
Institute for Applied Cryptography, ETH Zurich
Published: March 2026

Abstract

We present a systematic analysis of OAuth2 token refresh race conditions affecting 78% of web applications using rotating refresh tokens. Our research identifies a novel attack vector - the Concurrent Refresh Token Replay (CRTR) attack - enabling session hijacking.

1. Introduction

The OAuth2 specification (RFC 6749) recommends refresh token rotation as a security measure. However, the interaction between token rotation and multi-tab browser usage creates a fundamental race condition.

2. Threat Model

We consider an attacker who has obtained a single refresh token. The goal is to maintain persistent access even after the legitimate user refreshes tokens.
